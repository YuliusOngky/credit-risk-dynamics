CAP REST API

Integrate credit assessment, evidence upload (including manual SLIK), and decision governance into your core systems. Base URL: https://api.cap.creditriskdynamics.com

Endpoints

  • Authentication
  • e-KYC Parsing
  • SLIK Upload Analysis
  • Credit Scoring
  • Early Wage Access
POST /api/v1/auth/token

Authentication

Obtain a short-lived bearer token using your institution client credentials. Tokens expire after 3600 seconds and must be sent in the Authorization header for all subsequent requests.

Request body

ParameterTypeDescription
client_idstringIssued during CAP onboarding
client_secretstringSecret key (rotate via admin portal)
grant_typestringMust be client_credentials
cURL
curl -X POST https://api.cap.creditriskdynamics.com/api/v1/auth/token \
  -H "Content-Type: application/json" \
  -d '{
    "client_id": "cap_live_bpr_001",
    "client_secret": "••••••••••••",
    "grant_type": "client_credentials"
  }'
Response 200 OK
{
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "token_type": "Bearer",
  "expires_in": 3600,
  "scope": "assessment:write kyc:read evidence:write"
}
POST /api/v1/kyc/parse

e-KYC Parsing

Submit identity document images or vendor payloads for structured extraction. Parsed fields are attached to the decision record and validated against OJK e-KYC guidelines.

Request body

ParameterTypeDescription
application_idstringCAP application reference
document_typestringktp, npwp, or selfie
document_base64stringBase64-encoded image (max 5MB)

curl -X POST https://api.cap.creditriskdynamics.com/api/v1/kyc/parse \

    "document_base64": "iVBORw0KGgo..."

POST /api/v1/evidence/slik-upload-analysis

SLIK Upload Analysis

Upload a SLIK report that your team obtained manually from the OJK SLIK portal (PDF or official export file). CAP does not call SLIK providers directly. After upload, CAP analyzes the file—extracts summary fields, validates format, and attaches immutable evidence to the decision record.

Request body

ParameterTypeDescription
application_idstringCAP application reference
file_base64stringSLIK PDF or export file (max 10MB)
file_namestringOriginal filename for audit (e.g. slik-andi-prasetyo.pdf)
nikstringMust match applicant NIK on the report
uploaded_bystringAnalyst or system user ID
portal_pull_datestringDate the report was downloaded from SLIK (ISO 8601)
cURL
curl -X POST https://api.cap.creditriskdynamics.com/api/v1/evidence/slik-upload-analysis \
  -H "Authorization: Bearer $CAP_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "application_id": "APP-2026-88421",
    "file_name": "slik-andi-prasetyo.pdf",
    "file_base64": "JVBERi0xLjQK...",
    "nik": "3174010101990001",
    "uploaded_by": "analyst.joko",
    "portal_pull_date": "2026-05-11"
  }'
Response 200 OK
{
  "evidence_id": "EV-SLIK-2026-004512",
  "application_id": "APP-2026-88421",
  "ingest_status": "parsed",
  "source": "manual_portal_upload",
  "extracted_summary": {
    "collectibility": "1",
    "total_outstanding_idr": 125000000,
    "active_facilities": 3,
    "worst_dpd_12m": 0
  },
  "evidence_uri": "cap://records/APP-2026-88421/evidence/slik-004512",
  "recorded_at": "2026-05-11T10:22:41Z"
}
POST /api/v1/assessment/evaluate

Credit Scoring

Run a governed credit assessment using the policy version registered for your institution (tenant). CAP applies that tenant's rules, thresholds, and scorecards—not a single global model—then returns score, risk grade, policy checks, and an AI recommendation snapshot for committee review.

Custom scoring per tenant

Each financial institution (tenant) maintains its own scoring configuration in CAP: product lines, cut-offs, weightings, and policy bundles. The policy_version field selects which approved ruleset to execute (for example RETAIL-UNSECURED-v3.2 for Bank A vs MICRO-SME-v1.4 for a BPR). The same API endpoint serves all tenants; isolation and auditability are enforced in the governance layer.

Inputs include applicant profile, uploaded SLIK summary, and income evidence already on the decision record.

Request body

ParameterTypeDescription
application_idstringUnique application ID
policy_versionstringApproved tenant ruleset ID (custom scoring per institution). Example: RETAIL-UNSECURED-v3.2
requested_amountnumberLoan amount in IDR
tenor_monthsintegerRequested tenor
applicantobjectProfile, employment, income fields
cURL
curl -X POST https://api.cap.creditriskdynamics.com/api/v1/assessment/evaluate \
  -H "Authorization: Bearer $CAP_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "application_id": "APP-2026-88421",
    "policy_version": "RETAIL-UNSECURED-v3.2",
    "requested_amount": 75000000,
    "tenor_months": 24,
    "applicant": {
      "monthly_income": 18500000,
      "employment_type": "permanent"
    }
  }'
Response 200 OK
{
  "decision_record_id": "CDR-2026-000184",
  "application_id": "APP-2026-88421",
  "score": 712,
  "risk_grade": "B+",
  "policy_checks": {
    "passed": 7,
    "failed": 1,
    "details": ["DTI within limit", "SLIK collectibility OK"]
  },
  "recommendation": {
    "action": "approve",
    "max_amount_idr": 75000000,
    "model_version": "cap-score-2.4.1"
  },
  "evaluated_at": "2026-05-12T11:02:08Z"
}
GET /api/v1/ewa/eligibility

Early Wage Access

Check payroll-linked eligibility for earned wage access products. Uses employer verification and CAP risk limits without initiating a full loan application.

Query parameters

ParameterTypeDescription
employee_idstringEmployer HRIS employee reference
employer_idstringCAP registered employer ID
cURL
curl -G https://api.cap.creditriskdynamics.com/api/v1/ewa/eligibility \
  -H "Authorization: Bearer $CAP_TOKEN" \
  --data-urlencode "employee_id=EMP-44821" \
  --data-urlencode "employer_id=EMPLOYER-BNI-001"
Response 200 OK
{
  "eligible": true,
  "max_advance_idr": 4200000,
  "accrued_wages_idr": 8400000,
  "advance_limit_pct": 50,
  "next_payroll_date": "2026-05-28",
  "risk_tier": "low"
}
